# ターゲット指定システム

E2E テストの実行対象を柔軟に指定するための項番システム。

## 項番の構造

項番は `.` で区切られた 3 つの数字で構成される。

```
{scenario}.{case}.{step}
```

| 位置   | 意味     | 対応する spec.json フィールド                                                         |
| ------ | -------- | ------------------------------------------------------------------------------------- |
| 1 桁目 | scenario | `scenarios` のインデックス（1 始まり）                                                |
| 2 桁目 | case     | トップレベル `case.rows` もしくは `scenarios[*].case.rows` のインデックス（1 始まり） |
| 3 桁目 | step     | `scenarios[*].steps` のインデックス（1 始まり）                                       |

例: `2.3.1` は「scenario 2 × ケース row 3 × step 1」を指す。

## 基本的な指定方法

### 1. 単一項目指定

```bash
--target 1.1.1
```

特定の 1 項目のみを実行する。

### 2. 複数項目指定（コロン区切り）

```bash
--target 1.1.1:1.1.2:1.2.1
```

指定した複数の項目を実行する。

### 3. ワイルドカード指定

```bash
--target 1.*.*     # scenario1のすべてのテスト
--target 1.2.*     # scenario1, case2のすべてのstep
--target *.1.*     # すべてのscenarioのcase1
```

`*` を使用して部分的に一括指定する。

### 4. 範囲指定

```bash
--target 1.1.1-5       # case1, scenario1のstep1からstep5まで
--target 1.1-3.*       # case1のscenario1からscenario3まですべて
--target 1-3.*.*       # case1からcase3まですべて
```

`-` を使用して範囲を指定する。

### 5. 複合指定（カンマ区切り）

```bash
--target 1.1.1-3,2.*.*,3.2.1:3.2.5
```

上記の指定方法を `,` で組み合わせて複数指定する。

## ターゲット解析の処理手順

`--target` が指定されている場合、次のように処理し。

1. ターゲット解析: 指定されたターゲット文字列を解析し、実行対象の項番リストを生成する。
2. 項番マッチング: 仕様から生成される全項番と、ターゲット項番リストを比較してマッチするもののみを実行対象とする。
3. 実行順序: マッチした項番を項番順（昇順）に並べて実行する。

### 解析処理の詳細

1. カンマ分割: ターゲット文字列を `,` で分割してグループ化
2. コロン分割: 各グループを `:` で分割して個別指定を展開
3. 範囲展開: `-` を含む指定を範囲として展開
4. ワイルドカード展開: `*` を含む指定を実際の項番リストにマッチさせて展開
5. 重複除去: 展開後の項番リストから重複を除去

### 実行対象の制限

- 指定された項番が仕様に存在しない場合は警告を出力するが、存在する項番のみを実行する
- ターゲット指定により一部のテストをスキップする場合、スキップしたテストは結果レポートに含めない
- シナリオの途中からテストを開始する場合でも、該当シナリオの初期化処理（セッションクリア等）は必ず実行する

## 具体例

### 基本例

```bash
# 特定の1ステップのみ実行
--target 1.2.3

# 複数ステップを個別指定
--target 1.1.1:1.1.3:2.2.1

# シナリオ全体を実行
--target 1.1.*:1.2.*

# ケース全体を実行
--target 2.*.*
```

### 範囲指定例

```bash
# ステップ範囲
--target 1.1.1-5        # 1.1.1, 1.1.2, 1.1.3, 1.1.4, 1.1.5

# ケース範囲
--target 1.1-3.*        # 1.1.*, 1.2.*, 1.3.*

# シナリオ範囲
--target 1-3.*.*        # 1.*.*, 2.*.*, 3.*.*
```

### ワイルドカード例

```bash
# すべてのシナリオの最初のケース
--target *.1.*

# 特定シナリオのすべてのケース
--target 2.*.*

# すべてのシナリオの最初のステップ
--target *.*.1
```

### 複合指定例

```bash
# 複数パターンの組み合わせ
--target 1.1.1-3,2.*.*,3.2.1:3.2.5

# 展開結果:
# 1.1.1, 1.1.2, 1.1.3  (範囲指定)
# 2.1.1, 2.1.2, 2.2.1, 2.2.2, ...  (ワイルドカード)
# 3.2.1, 3.2.5  (個別指定)
```

## 実行制御への影響

### シナリオ初期化

シナリオの途中からテストを開始する場合でも、該当シナリオの初期化処理は必ず実行され。

```bash
# 例：1.2.3から開始する場合
--target 1.2.3

# 実行される処理：
# 1. シナリオ1の初期化（セッションクリア等）
# 2. ステップ1.2.3のみ実行
# 3. シナリオ1の後続ステップ・ケースはスキップ
```

### 結果レポート

ターゲット指定時の結果レポートには、実際に実行された項番のみが含まれ。

```bash
--target 1.1.*:2.2.1

# 結果レポートに含まれる項番：
# 1.1.1, 1.1.2, 1.1.3, ..., 2.2.1

# 含まれない項番：
# 1.2.*, 2.1.*, 2.2.2, ...
```

## エラーハンドリング

### 形式エラー

```bash
# 不正な文字
--target 1.1.a          # エラー：数字以外の文字

# 不正な記号
--target 1..1           # エラー：連続したドット

# 不正な範囲
--target 1.1.5-2        # エラー：逆順範囲
```

### 存在しない項番

```bash
--target 1.5.1          # 警告：項番1.5.1は存在しない（スキップ）
--target 1.*.*:99.1.1   # 警告：項番99.1.1は存在しない（1.*.*のみ実行）
```

### パフォーマンス考慮

大量の項番が指定される場合の最適化について説明し。

```bash
# 効率的：具体的な指定
--target 1.1.1:1.1.3:1.2.2

# 非効率的：過度なワイルドカード
--target *.*.*          # 全項目実行と同じ（ターゲット指定不要）
```
